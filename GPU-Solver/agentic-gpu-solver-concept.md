## 한 줄 정의
GPU 챌린지를 자동으로 통과시키고, 나아가 **속도 상위권**까지 끌어올리는 멀티-LLM 솔버. 신규 문제·사내 과제로 일반화 가능하게.

## 아키텍처 골격
- **하네스 = Hermes** (실행·툴호출·루프 제어). 그 위에 LLM 백엔드를 교체 가능하게 꽂음.
- **LLM = 순차 폴백 3단 사다리** (속도/정확성 우선). 퓨전은 미래 옵션으로 인터페이스만 열어둠.
- **코드봇 형태 = 모델 비종속 오케스트레이터** (단일 모델 플러그인 아님). 각 LLM을 공통 규격 어댑터로 감쌈.
  - 구현 = **오케스트레이터 + (추후)RAG**, 파인튜닝 없음. MVP는 RAG 없이, 2차에 RAG 도입.
  - 플러그인/MCP = "기존 에이전트 확장" 방식, 고도화 단계 선택적 부착.
- **측정·채점 = LeetGPU 원격 환경** 의존. 로컬 GPU 불필요(외부 API 전제).
- **지식 관리 = Obsidian**(사람용 검수) + 정규화 DB/인덱스(에이전트 질의용). frontmatter 스키마 공유.

## 수집 소스 (정정됨)
- **AlphaGPU = 답안 아님. 검증 토대.** 각 challenge 폴더 구성:
  - `challenge.html` = 문제 설명
  - `challenge.py` = **레퍼런스 구현 + 테스트 케이스 + 메타데이터** → 정확성 게이트의 실체
  - `starter/` = 프레임워크별 빈 스텁("그냥 pass") → 답안 가치 없음
- **개인 GitHub 레포 = 실제 답안 소스 (1차)** → 맥락·커밋 히스토리
- **LeetGPU 사이트 제출 = 답안 다양성 (2차)** → 런타임은 Pro 필요, 크롤링 보류
- 실제 답안은 *개인 레포 + 사이트 제출*에서만 옴 (AlphaGPU엔 없음)

## ⚠️ 라이선스 경고 (사내 활용 직결)
- **AlphaGPU 레포 = CC BY-NC-ND 4.0**: 상업적 사용 / 재배포 / 2차 저작 **금지**
  - NC → **"사내(상업) 프로젝트 사용" 목표와 충돌**
  - ND → 변형·파생물 제한
- 학습·개인연구는 회색지대, **사내 상업 활용은 별도 검토 또는 AlphaGPU 허가 필요**
- 개인 레포는 각자 라이선스 다름 → 수집 시 라이선스 메타 기록 필수
- **우회안(C): AlphaGPU 의존 최소화** = challenge.py를 베끼지 않고, 문제 개념에서 *독립적으로* 레퍼런스 구현 + 테스트 생성기를 자체 제작 (알고리즘 자체는 저작권 대상 아님)
- (법률 자문 아님 — 실제 활용 전 확인 권장)

## 핵심 루프 (2단 게이트)
1. **정확성 통과 게이트** (수집 코드도 자체 재검증)
2. **속도 토너먼트** (Pro의 percentile·Perfetto 트레이스로 병목 지목 → 다음 프롬프트 주입)
- 독창성 = **near-duplicate만 차단**, 나머지는 속도 최적화로 자연 분화

## 폴백 사다리 (순차 / 속도·정확성 우선)
- **코딩 루프 = API 모델만** (단일 API로 깨끗하게). 구독+Claude Code 경유는 루프에 안 넣음
  (Hermes 컨텍스트 재주입·세션 망각 마찰 회피).
- **Tier 1 (단일 1차)**: DeepSeek 또는 Codex 중 하나 — *초반 가벼운 테스트로 더 잘 푸는 쪽 결정*
- **Tier 2 (교차)**: Tier 1에서 안 쓴 나머지(Codex/DeepSeek)로 재시도
- **Claude = 코딩 루프 제외**, 비코딩(기획·문서·검수·디버깅분석)에만. 필요 시에만 직접 API로 한정 합류(선택).
- **승급 트리거**: 정확성 반복 실패 / 컴파일 반복 실패 (= 풀이 과정 속도 기준)
- 각 문제의 "어느 Tier가 풀었나 / 어디서 막혔나" 로그 적립 → 추후 라우팅·퓨전 근거

## "속도" 3종 구분 (혼동 방지)
1. **풀이 속도(과정)**: 통과까지 걸리는 시간/비용 → 폴백 사다리가 관리
2. **커널 런타임 속도(결과물)**: 생성 코드의 실행 속도 → 리더보드·percentile = **최종 목표 "상위권"**
3. LLM 토큰 출력 속도 → 거의 무시

## 최적화 정책 (토큰 절약)
- **한 방에 통과한 문제 → 즉시 속도 토너먼트(②) 진입**
- **고생해서 통과한 문제 → 통과로 만족, 2차 최적화 대기열에 보류(버리지 않음)**
- "쉽게/어렵게 통과" 판정 기준(시도 횟수 등)을 로그에 자동 기록 → 나중에 대기열 재소환

## 로드맵
- **MVP (1차)**: Easy 문제 + 개인레포 답안 + challenge.py(또는 자체) 검증 + 정확성 통과 루프 + 최소 하네스
  - 성공 기준: **목표 90% / 합격선 70~80% + 실패 로그 수집**
  - 제외(2차로): 사이트 크롤링, 속도 최적화, 원리 노트, Pro, 퓨전
- **2차**: Pro 전환, 속도 최적화 루프, 원리 노트 도입, 사이트 수집 합류, Medium/Hard 확장
- **장기**: 로컬 LLM 이전(이때만 GPU 필요), 모델 퓨전, 사내 프로젝트 이식

## 보조 도구
- **Godbolt** = 컴파일/PTX·SASS 디버깅 보조 (필수 아님, 속도 막힐 때)
- **패턴 노트** = solution에서 추출·추상화 (RAG 1차 검색 대상)
- **원리 노트** = 이론 기반, 속도 단계에서 수동 큐레이션 도입 (패턴 노트와 출처 다름, 통과 단계엔 생략)
- **Discussion(문제 오류)** = MVP에선 무시, 특정 문제 안 풀릴 때만 수동 참고

## 정규화 스키마 (방향)
- 답안 레코드 키 = **(문제 × 칩/GPU 타입 × 프레임워크)** — AlphaGPU는 CUDA·Mojo·Python 등 다중 프레임워크 지원
- 필드(잠정): 문제ID / 출처 / 코드 / 프레임워크 / 칩타입 / 라이선스 / 검증상태 / 런타임 / 기법태그
- **`no_solution` = 정식 상태** (특정 문제×칩 조합은 답 없음 → 빈칸 아니라 명시)
- 스키마 최종 확정은 *실제 레포 몇 개 열어본 뒤* (필드 사후 정착)
- 문제 매핑 = 에이전트 유사도 자동 매핑 → 사람 검수

## 고도화 묶음 (전부 2차 이후 보류)
